## 发布正文

Claude Code 很强，但很多人第一次用都会遇到一个问题：

它每次开新会话，都像一个刚入职的新同事。

你明明说过项目用 `pnpm`，它还是写 `npm`；
你讲过目录结构，下次它又重新猜；
你提醒过提交前要跑测试，换个会话又忘了。

这时候不要只在聊天里反复纠正，应该把规则写进 `CLAUDE.md`。

我建议先写 5 类内容：

1. 项目技术栈：语言、框架、包管理器
2. 常用命令：安装、启动、测试、构建
3. 目录边界：什么代码应该放哪里
4. 必须遵守：团队约定、提交前检查
5. 参考入口：架构文档、API 规范、测试示例

但注意，`CLAUDE.md` 不是项目百科。

每次都会影响 Claude 判断的，写进去；
只在某类文件里用到的，拆到 `.claude/rules/`；
本地环境信息，放 `CLAUDE.local.md`；
很长的背景资料，放 `docs/`，只留入口。

真正好用的 AI 编程，不是让 AI 猜你的项目，而是提前把项目规则交给它。

建议收藏，下次开新项目直接照这个结构写。
