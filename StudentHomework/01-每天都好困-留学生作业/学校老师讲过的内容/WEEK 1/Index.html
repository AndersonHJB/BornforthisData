2. HTML

Boilerplate Code

-The boilerplate code above, which we set up in week 1, creates the basic structure for a web page. Type it out exactly as you see it here in the browser. You will only ever have to do this once if you copy and paste for your records. 'Save as' right now and create a 'boilerplate.html' file and you will always be ready to go in future.

-The <!DOCTYPE html> tag tells browsers that our document contains html code.

-The <html> tag specifics where our html starts and stops - notice the tag is 'closed' with a </html> tag at the end of the document - the forward slash indicates a close tag. Almost all html tags work in this way, with an opening and closing tag.

-The <head> tag specifics a section of code which is loaded before the rest of the page - remember the browser will load tags in the order you write them, top to bottom, left to right. We can link to other pieces of code such as CSS and Javascript [we will learn about these later] and provide crucial details that will usually affect your entire page.

-The <meta> tag includes an attribute 'charset' - or character set - which is set to UTF-8, the standard for contemporary websites, ensuring all text characters are rendered properly. The name and content attributes tell the browser to render the page at the available width of the viewport, which is the default size of the browser window. Later on we will learn to make our sites 'responsive', which means they will respond to the actual size of the broweser window.

-The <title> tag specifies the title of the page in the browser tab - for example 'WEEK 1' here, as you can see in the tab name in your browser.

-The <body> tag specifies the body or substance of the page itself and is where the rest of your code goes.

HTML Elements

-A html element is one piece of an HTML page - defined by a start tag and end tag and everything in between, such as any of the above.

-HTML elements can be inside - and therefore included in - other elements - this is called nesting.

Formatting

-It is standard practice to use lowercase tags and to indent one tab or space to show the relationship between parent and child elements - as you can see in the boilerplate code above.

-People get very passionate aboute using spaces or tabs - just pick one and be consistent. Again if you control click and choose 'view source' you will be able to see how the code in this page is formatted so it is easy to tell which elements are nested without others. It is important that you keep your code well organised and readable - you will thank yourself later.

5. INTRODUCTION TO CSS

As a prelude to next week, let's try some very simple CSS. I have applied some styling to the heading immediately above. If you 'view source' you will see some extra code inside the opening h1 tag style="....".. and so on.

CSS stands for Cascading Style Sheets and is used to style HTML elements. It allows us to change the appearance of our web pages, including colors, fonts, layouts, and more.

CSS is a separate language from HTML, but it works closely with it. We can write CSS directly in our HTML files using the style attribute, or we can link to an external CSS file.

CSS can be applied in three different ways: inline, internal, and external. Today we will focus on inline styling.

To style any element you need to put CSS inside the opening tag. Following the syntax "style="Property: Attribute" - if there are multiple properties, seperate with a semi-colon; Style lets us control how an element will appear - for example style="color:red;" and style="background-color:red;" - see the heading in the source in this section for an example.

We can also use the style attribute to position elements, change their size, and apply other effects. We will get further into formatting and CSS next week. Today we are just experimenting.
