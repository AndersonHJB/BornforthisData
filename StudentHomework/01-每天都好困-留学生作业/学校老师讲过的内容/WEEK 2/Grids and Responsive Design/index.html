How to insert CSS

HINT: Have this site open in your browser as you code today.

Cascading style sheets (CSS) is a language used to control the styling of HTML documents, controlling how elements should be displayed.

There are three ways to add CSS to an HTML document - an internal style sheet, an external style sheet and inline styling. For the most part, we will be using EXTERNAL styling.

Inline styling is done by using the style attribute in your html tags. We did this last week.

External style sheets are text files that you link to from the head of a HTML document - as you can see if you view source on this document or look at the [link] tags in the [head] of your boilerplate code.

To create an internal style sheet you use the <style> tag in your html document and write your CSS there using the same syntax as an external stylesheet. Mostly you will find this method is not necessary.

Cascading style sheets are so named because the styling details cascade into a 'virtual' style sheet, with the most recent style rules processed employed for each element, so long as they follow this order of priority:

1. Inline style

2. External and internal style sheets - the hierarchy is unclear here so use one or the other but not both.

3. Browser defaults

Selectors

See here for a full list of selectors.

The following can all be used as selectors:

If you want to style every instance of particular HTML element - for example <h1>, <p>, <div>, etc - they can be selected in the following way: elementname {property:value;}

i.e. "h1. {color:blue; font-size: 2em;}" will make all h1 elements blue and 2em in size.

Likewise if you want to select all <div> elements you can use the selector div {property:value;} - this is how the CSS for this page is written.

Classes: If you want to make a more refined selection, use CLASSES - put - class="classname" inside the opening tag of an element. These are then selected in CSS in the following way: ..classname {} - look in the source to see how classes and ids are applied to this div, for instance.

HINT: In this page you can see how I have used two classes for each DIV. 'section' is for styling that applies to all DIVs, while 'col-1-12' determines width. You simply write multiple classes in side the opening tag. i.e. class="sections col-6"

Ids - Individual elements can be selected using an ID. These are selected in CSS in the following way: #tagname {}

View source on this document for examples - notice that you can also use the universal selector '*' to select multiple elements of a particular class - see the css for columns in this page for instance.

Colours - HTML supports 140 standard colour names but colours can also be set using RGB values and HEX values - view source for examples and remember there are many colour pickers online you can use to select individual colours and even whole colour schemes. I have used this site again.

Text, Fonts and Links

CSS is also used to style text and links and control fonts.

Text is most commonly coloured, aligned and decorated.

The same is true for links but because links involve different states we use some further variants on selectors:

a:link - a normal, unvisited link

a:visited - a link the user has visited

a:hover - a link when the user mouses over it

a:active - a link the moment it is clicked

Fonts are set using the @font-face selector to define a specific font, you can download fonts and link locally, ensuring they will always displya correctly, or you can link to fonts on the web, they will usually have a .ttf extension. Feel free to experiment and change the font of your site - see here for free type faces.

You can use generic font-families such as Times or Georgia, which will be on your users computers, or font families such as such as serif and sans-serif. Alternatively you can use online fonts, such as those from google fonts, which provides you with code to put in your header that will link to the font, and css which you can apply to individual elements.

You can then also specify font-size using either px - pixels - or em - or percentages. em is a measure of the font-size relative to the default font size set on the user's computer - view source on this document to see examples. Using em or percentage is advisable as this will ensure responsivity - more about that later

What is Responsive Web Design?

Responsive web design involves using HTML and CSS to automatically resize, and even re-layout, a website so that it looks good on all devices.

For many kinds of websites it is smart to design 'mobile first' - this means that you design so the site will work well on a mobile device first - because most people will visit it on a mobile device - and then add responsivity to handle other use cases.

The decisions you make here will depend on your website and your particular idea - for instance, your site may be designed to be unresponsive or to only work in a browser for artistic reasons.

Making your page responsice is relatively simple - there are few fail-safe rules to follow.

Setting the Viewport

Make sure you use the following code in your header when making responsive pages (your boilerplate code includes this):

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This allows the browser to handle the pages dimensions and layout based on device screen size.

See here for an explanation of this property.

Designing with Grids

Using a grid-view to design your site can help you manage complex layouts and responsivity. You have already created a grid style layout when you copied the code at the beginning of the tutorial.

When using a grid-view the page is divided vertically into a series of columns - often twelve - which are then organised into horizontal rows.

This gives you a way to measure and proportion segments of your page.

As you can see in the CSS we have 12 columns with width percentages that TOGETHER add up to 100%.

We then simply use divs to arrange the page - We mark horizontal divs with the 'row' class, and then the divs within it must add up to 100% - we might mark 4 x divs as 'col-3', for example.

Media Queries and Breakpoints

Media queries use the @media rule, as you can see here, to check on screen size and add 'breakpoints' to change the CSS based on the new size:

        
            @media only screen and (max-width: 1080px) {
                        body {
                                background-color: blue;
                                }
                            }

As you can see in the cource code, when the window size is reduced to 1080px or below (common phone resolution) a series of rules are changed - columns all become 100% width, the position and display of the hamburger changes so it will not be left off the page or overlap with the heading. The DIVS are all simply displayed as a single row, which makes them readable on a small screen.

Responsive Text, Images and Videos

Text, images and videos can all be made to be responsive too.

Font-size can be made responsive by using vw - viewport width - instead of em or px - 1vm is 1% of the viewport width (view source for an example and note I've deliberately chosen to use a mix of vw and em).

Images can be made responsive by setting the width to 100% and the height to auto like this:

        
                img {
                      width: 100%;
                      height: auto;
                    }
